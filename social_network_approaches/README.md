# Обнаружение контрафактных товаров на Ozon с использованием методов детекции фейков в социальных сетях

Задача обнаружения контрафакта на маркетплейсе структурно похожа на задачу детекции фейковых новостей в социальных сетях: в обоих случаях нужно найти неаутентичный мультимодальный контент среди огромного потока подлинного, используя текстовую и визуальную информацию. Фундаментальная гипотеза, объединяющая оба домена: фейковый контент характеризуется семантическим несоответствием между текстом и изображением. В социальных сетях создатели фейков используют нерелевантные изображения, чтобы привлечь внимание к ложным утверждениям; в e-commerce контрафактные продавцы аналогично используют фотографии оригинального товара с подменённым описанием. В данной работе мы адаптируем методы из четырёх статей по детекции фейков в соцсетях и исследуем их применимость к задаче контрафакта.

## Условия экспериментов

Все эксперименты выполняются на единой тестовой выборке из 19 818 товаров, сформированной **seller-based разбиением** (random_state = 42). Пересечение множеств продавцов между обучающей и тестовой выборками строго равно нулю — это воспроизводит реальный deployment-сценарий, в котором модель оценивает товары продавцов, ни разу не встречавшихся в обучении. Дисбаланс классов — 1:14.

Ключевая бизнес-метрика — **Recall@P ≥ 0,9** (полнота при ограничении точности не ниже 0,9). Дополнительные метрики — ROC-AUC и PR-AUC.

## Модели

### 1. SpotFake-Ozon

**Статья:** Singhal et al. *SpotFake: A Multi-modal Framework for Fake News Detection*. ACM MM, 2020.

**Идея:** каждая модальность обрабатывается независимым энкодером, затем представления конкатенируются и подаются в классификатор (late fusion). Авторы оригинальной статьи отказываются от вспомогательных подзадач (как у EANN или MVAE) и показывают, что standalone-классификатор на late fusion превосходит обе системы на Twitter MediaEval и Weibo.

**Архитектура:**

```
Текст (768)    ──► TextEncoder  ──► text_emb (128)
                                                      \
Картинка (512) ──► ImageEncoder ──► img_emb  (128) ──► Concat (384) ──► Classifier ──► sigmoid
                                                      /
Таблица (37)   ──► TabEncoder   ──► tab_emb  (128)
```

Каждый `ModalityEncoder`: `Linear → LayerNorm → GELU → Dropout → Linear → LayerNorm`. Классификатор: 384 → 256 → 64 → 1 с Dropout(0,3) и GELU. Суммарное число обучаемых параметров — 789 761.

**Адаптации под нашу задачу:**

| Параметр | SpotFake (оригинал) | SpotFake-Ozon |
|---|---|---|
| Домен | Фейковые новости в соцсетях (Twitter, Weibo) | Контрафакт на Ozon |
| Текстовый энкодер | BERT-Base (англ.) → 32 | ruBERT (рус., DeepPavlov) → 128 |
| Визуальный энкодер | VGG-19 (fine-tune) → 32 | CLIP ViT-B/32 (готовые эмбеддинги, 512) → 128 |
| Модальности | 2 (текст + картинка) | 3 (+ табличные признаки продавца) |
| Fusion | Конкатенация | Конкатенация |
| Loss | Binary Cross-Entropy | Focal Loss (γ=2, α=0,75) |
| Дисбаланс | Не учитывается | WeightedRandomSampler |
| Оптимизатор | Adam | AdamW + CosineAnnealingLR (30 эпох) |
| Датасет | Twitter / Weibo (сбалансированные классы) | Ozon eCup 2025 (1:14) |

### 2. SAFE-Ozon

**Статья:** Zhou et al. *SAFE: Similarity-Aware Multi-modal Fake News Detection*. PAKDD, 2020. [arXiv:2003.04981](https://arxiv.org/abs/2003.04981)

**Идея:** контрафакт часто использует картинку оригинального товара с другим текстом, или наоборот — копирует описание оригинала, но показывает другой товар. SAFE предлагает явно измерять **семантическое несоответствие между текстом и картинкой** и использовать его как дополнительный сигнал. Из всей архитектуры SAFE заимствуется только этот единственный компонент — скалярное косинусное сходство; нейросетевая часть SAFE (адаптивный автоэнкодер) не переносится, поскольку базовый каркас остаётся от SpotFake-Ozon.

**Что добавляем к SpotFake-Ozon:**

1. Получаем CLIP-эмбеддинг текста (поля `name_rus + description`) через `openai/clip-vit-base-patch32`.
2. Считаем косинусное сходство между CLIP текста и CLIP картинки:
   ```python
   similarity = F.cosine_similarity(clip_text_emb, clip_img_emb)
   ```
3. Добавляем `similarity` как 385-е измерение в финальный вектор классификатора.

**Эмпирическая проверка гипотезы:** среднее косинусное сходство для оригинальных товаров — 0,277, для контрафактных — 0,256 (Δ = 0,021 при σ ≈ 0,08). Разница статистически значима, но в масштабе стандартного отклонения составляет менее 0,3σ — на грани практической различимости.

### 3. SpotFake-Ozon-combined

**Идея:** расширение признакового пространства поверх SAFE-Ozon без изменения архитектуры. Разведочный анализ выявил статистически значимые различия между классами по длине текстовых полей листинга: контрафакт чаще характеризуется либо очень короткими описаниями (копирование без переработки), либо аномально длинными (SEO-заполнение).

**Что добавляем:**

- **Текстовые метрики листинга:** `description_length`, `name_length`, `brand_length`, бинарные флаги `has_description`, `has_brand`, а также логарифмические трансформации `log_description_length`, `log_name_length` для устранения правостороннего скоса.
- **LOO-кодирование бренда:** новый признак `brand_problem_rate` — доля контрафактных товаров среди всех товаров данного бренда. Корреляция с целевой переменной r = +0,42. Кодирование выполняется по схеме Leave-One-Out на 5 фолдах для предотвращения утечки таргета; для редких/новых брендов на тесте используется глобальное среднее.
- **Mean pooling в ruBERT:** вместо токена [CLS] используется взвешенное среднее скрытых состояний всех не-padding токенов с весами по маске внимания. Замена обоснована тем, что описания товаров — длинные многопредложенческие тексты, и при усечении до 128 токенов [CLS] теряет информацию из хвостовых частей.

Табличная ветка расширена с 37 до 57 признаков; скрытый слой TabEncoder расширен с 128 до 256 нейронов. Итоговый вектор: 384 + 1 (sim) = 385 измерений.

### 4. MOMENTA-Ozon

**Статья:** Pramanick et al. *MOMENTA: A Multimodal Framework for Detecting Harmful Memes and Their Targets*. EMNLP Findings, 2021.

**Мотивация:** архитектурный потолок late-fusion остаётся принципиальным — классификатор получает уже агрегированные модальные представления и не имеет возможности обучать тонкие cross-modal взаимодействия. MOMENTA предлагает структурированный механизм межмодального взаимодействия на трёх принципах: смесь экспертов (Mixture-of-Experts) для каждой модальности, двунаправленное перекрёстное внимание (Bidirectional Co-Attention) для семантического выравнивания, явная ветвь несоответствия (Discrepancy Branch), извлекающая структурированную информацию о расхождениях (масштаб и знак различий по каждому измерению).

**Архитектура (4 блока):**

1. **Модальные энкодеры с MoE.** Текст (ruBERT 768) и картинка (CLIP 512) проецируются в общее пространство emb_dim = 128, проходят LayerNorm и Multi-Head Self-Attention. Затем каждая модальность обрабатывается слоем MoE с K = 4 экспертами (двухслойные FFN с GELU и dropout); веса смешивания вычисляются гейтинговой сетью через softmax. Обоснование: контрафакт представлен принципиально различными паттернами (демпинг, имитация бренда, новые продавцы) — MoE позволяет специализировать экспертов на каждом из них.
2. **Bidirectional Co-Attention.** Текстовое представление используется как запрос к визуальному контексту и симметрично наоборот — явное моделирование семантического выравнивания между модальностями.
3. **Discrepancy Branch.** Конкатенация |M_text − M_img| и M_text ⊙ M_img фиксирует и масштаб, и знак расхождений; результат взвешивается обучаемым скалярным η.
4. **TemporalAggregator.** Листинги продавца упорядочиваются по временной метке и обрабатываются скользящим окном; модуль вычисляет EMA предсказаний и drift — отклонение от среднего. Мотивация: недобросовестные продавцы нередко начинают с легитимных товаров и постепенно вводят контрафакт по мере роста рейтинга.

**Многоцелевая функция потерь** (7 компонентов):

| Компонент | Назначение |
|---|---|
| L_focal | Основная классификация, фокус на трудных примерах |
| L_align | Штраф за низкое выравнивание текст ↔ картинка в Co-Attention |
| L_contrastive | InfoNCE: сближение представлений одного класса |
| L_temporal | Согласованность предсказаний продавца во времени |
| L_rdrop | KL-дивергенция между двумя проходами через dropout |
| L_domain | Domain-adversarial: инвариантность к товарной категории |
| L_proto | Прототипическое обучение: расстояние до центроидов классов |

### 5. MMD (адаптация MMD-Thinker)

**Статья:** Tan et al. *MMD-Thinker: Adaptive Multi-Dimensional Thinking for Multimodal Misinformation Detection*. 2024.

**Идея:** иерархия трёх режимов мышления — реактивный (быстрое решение по поверхностным признакам), семантический (анализ текста) и перспективный (глубокое рассуждение с визуальной модальностью). Адаптивный механизм выбора режима (ModeGate) определяет, какой уровень задействовать для конкретного объекта.

#### Почему прямой нейросетевой перенос не работает

Промежуточная реализация прямо переносила архитектуру MMD-Thinker — нейросеть с тремя головами и softmax ModeGate. Результат: **PR-AUC = 0,3824, Recall@P ≥ 0,9 = 0** (на 38,6% ниже CatBoost-бейзлайна). Три причины неудачи:

1. **Коллапс ModeGate.** Среди табличных признаков присутствовал `seller_problem_rate` (r с таргетом = 0,711). Уже на первой эпохе gate выучивал опираться исключительно на него; распределение весов режимов вырождалось в [1,00; 0,00; 0,00] и не менялось. Вся конструкция трёх голов сводилась к одномодальной табличной модели.
2. **Вырождение функций потерь.** Focal loss падал до 0,0000 к 10-й эпохе (переобучение на тривиальное решение), policy loss принимал отрицательные значения.
3. **Неадекватность нейросетей для табличных данных с сильным целевым кодированием.** Четыре признака LOO-кодирования (продавец, бренд, категория, наименование) дают CatBoost ROC-AUC = 0,9051 без манипуляций; нейросеть достигала лишь 0,7566.

#### Решение: ансамблевая реализация многорежимного мышления

Сохраняем концептуальную трёхрежимную иерархию MMD-Thinker, но реализуем её через ансамбль из трёх независимо обученных CatBoost-моделей и мета-модель на стекинге.

| Компонент MMD-Thinker | Реализация в MMD-модели |
|---|---|
| Mode 1: Quick response | CatBoost на 79 табличных признаках |
| Mode 2: Semantic analysis | CatBoost + 50 SVD-компонент TF-IDF (129 признаков) |
| Mode 3: Deep deliberation + Attribution | CatBoost + SVD + 32 PCA-компоненты CLIP (161 признак) |
| ModeGate (softmax) | Логистическая регрессия на OOF-предсказаниях (стекинг) |
| Mixed advantage A_sample + A_mode | Мета-признаки: предсказания + \|Mᵢ−Mⱼ\| + Mᵢ·Mⱼ |
| KL-penalty | L2-регуляризация (C = 0,5) |

**Двухуровневое обучение.** На уровне 1 каждый из трёх режимов обучается в схеме 5-fold StratifiedKFold; OOF-предсказания гарантируют отсутствие утечки таргета. На уровне 2 логистическая регрессия предсказывает целевую переменную на основе 9 мета-признаков (3 предсказания + 3 попарные разности + 3 попарных произведения). Взвешивание примеров: w_i = 1 + (1 − 2·|p_i^(1) − 0,5|) — объекты с наибольшей неопределённостью Mode 1 получают повышенный вес.

**Признаки целевого кодирования** (наиболее информативные):

| Признак | r с таргетом |
|---|---|
| seller_problem_rate | 0,711 |
| category_problem_rate | 0,563 |
| name_problem_rate | 0,507 |
| brand_problem_rate | 0,422 |

### SpotFake-Ozon

Late fusion даёт прирост по PR-AUC относительно лучшего бейзлайна CatBoost (+4,3 п.п.) и подтверждает ценность объединения трёх модальностей. Однако **Recall@P ≥ 0,9 = 0**: WeightedRandomSampler в комбинации с Focal Loss смещает выходное распределение в узкий диапазон [0,4; 0,6], в котором отсутствует область высокой уверенности. Модель «знает», что контрафакт существует, но не способна уверенно указать на конкретный экземпляр.

### SAFE-Ozon

Введение скалярного признака `similarity` восстанавливает Recall@P ≥ 0,9 до 0,0838 — впервые в нейросетевой линейке достигается ненулевая полнота при P ≥ 0,9. Аблация показывает противоречивый результат: прямой вклад `sim_i` на инференсе пренебрежимо мал (обнуление меняет PR-AUC лишь на 0,0001), однако наличие признака **на этапе обучения** меняет траекторию оптимизации и приводит к более калиброванному распределению выходных вероятностей. Признак SAFE действует не как самостоятельный предиктор, а как регуляризатор обучения.

### SpotFake-Ozon-combined

Лучший результат среди нейросетевых экспериментов с late-fusion-каркасом. Прирост R@P ≥ 0,9 относительно SAFE-Ozon — +0,043 (рост в 1,51 раза); относительно CatBoost-бейзлайна — рост в 8,7 раза.

| Конфигурация | ROC-AUC | PR-AUC | ΔPR-AUC vs полная |
|---|---|---|---|
| Полная SpotFake-Ozon-combined | 0,9168 | 0,6939 | — |
| Без NaN-флагов | 0,8801 | 0,6303 | −0,064 |
| Без log-трансформаций | 0,8956 | 0,6543 | −0,040 |
| Без ratio features | 0,8929 | 0,6819 | −0,012 |
| Без seller_problem_rate | 0,8931 | 0,6830 | −0,011 |

Наибольший вклад вносят **бинарные NaN-флаги** (исключение снижает PR-AUC на 5,3 п.п.) и **логарифмические трансформации** (−2,9 п.п.).

### MOMENTA-Ozon

Отрицательный результат: PR-AUC = 0,5562, R@P ≥ 0,9 = 0,0323 — **ниже всех более простых мультимодальных моделей**. Архитектура потенциально обладает большей предсказательной ёмкостью, но требует существенно большего объёма обучающих данных или более жёсткой регуляризации для устойчивого обобщения на seller-based-разбиении. На имеющемся объёме усложнение нейросети контрпродуктивно.

| Конфигурация | ROC-AUC | PR-AUC |
|---|---|---|
| MOMENTA-Ozon (полная) | 0,8750 | 0,5562 |
| Без табличных признаков | 0,8378 | 0,4480 |
| Без Discrepancy Branch | 0,8074 | 0,4275 |

Ablation подтверждает теоретическое обоснование MOMENTA: **Discrepancy Branch — наиболее значимый архитектурный компонент** (ΔPR-AUC = +0,1287). Тем не менее даже с учётом этого вклада итоговое качество уступает SpotFake-Ozon-combined.

### MMD-модель

**Mode 3 — лучший результат среди всех экспериментов:** PR-AUC = 0,7579 (на 13,5 п.п. выше базового CatBoost), Recall@P ≥ 0,9 = 0,1181 (в 8,1 раза выше CatBoost-бейзлайна).

Поэтапное добавление модальностей даёт устойчивое улучшение по R@P ≥ 0,9: Mode 1 → Mode 2 (+текст): +0,0703 п.п.; Mode 2 → Mode 3 (+картинка): +0,0367 п.п. Несмотря на то что визуальные признаки вносят наименьший абсолютный прирост PR-AUC, их вклад в R@P ≥ 0,9 сопоставим с текстовым: изображение несёт информацию, ортогональную тексту (упаковка, маркировка, качество фото не отражены в текстовых полях).

**Поведение мета-модели:** высокий ROC-AUC (0,9437) и PR-AUC (0,7344), но R@P ≥ 0,9 = 0,0004 — логистическая регрессия оптимизирует log-правдоподобие и не фокусируется на области высокой точности PR-кривой. Для максимизации R@P ≥ 0,9 оптимально использовать Mode 3 напрямую.

## Выводы

1. **Late fusion работает, но требует калибровки.** SpotFake-Ozon даёт прирост PR-AUC, но коллапсирует на метрике R@P ≥ 0,9 из-за смещения распределения вероятностей. Признак SAFE действует как регуляризатор обучения и восстанавливает ненулевую полноту при P ≥ 0,9.

2. **Доменные признаки эффективнее архитектурного усложнения.** SpotFake-Ozon-combined (+20 табличных признаков) превосходит и SAFE-Ozon, и MOMENTA-Ozon. Бинарные NaN-флаги и log-трансформации длин текстов дают наибольший вклад.

3. **Усложнение нейросетевой архитектуры на ограниченном объёме данных контрпродуктивно.** MOMENTA и прямой перенос MMD-Thinker уступают значительно более простым моделям. Discrepancy Branch концептуально оправдан (вклад +0,1287 PR-AUC), но общая ёмкость MOMENTA приводит к переобучению.

4. **MMD-Thinker напрямую не переносим на табличные данные с сильным целевым кодированием.** ModeGate коллапсирует на доминирующий признак, focal/policy loss вырождаются, нейросеть проигрывает CatBoost на 16,4 п.п. ROC-AUC.

5. **Концепция многорежимного мышления успешно реализуется через ансамбль.** Mode 3 (CatBoost + TF-IDF SVD + CLIP-PCA) — лучший результат: PR-AUC = 0,7579, R@P ≥ 0,9 = 0,1181. Принципы адаптивного мышления переносятся в классический ансамблевый фреймворк при условии правильной идентификации функциональных аналогов теоретических компонентов.
