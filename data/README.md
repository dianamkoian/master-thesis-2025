# Данные

Раздел содержит описание данных, использованных в магистерской ВКР «Система контроля качества для e-commerce на основе кросс-доменной адаптации методов обнаружения фейков с использованием больших мультимодальных моделей» (К. У. Азимова, ВШЭ, 2026).

## Источник данных

Основной датасет — **Ozon eCup 2025** (трек «Контроль качества»), предоставленный организаторами соревнования.

- Официальная страница соревнования: <https://ecup.ozon.ru/>
- Объём: 197 198 товарных карточек, 11 167 продавцов
- Категории: электроника и оргтехника
- Дисбаланс классов: ~7,95 % контрафакта (positive class)

## Структура `ozon_train.csv`

Файл содержит 45 признаков на товарную карточку:

### Идентификаторы и target
| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | int | ID строки |
| `ItemID` | int | ID товара |
| `SellerID` | int | ID продавца (используется для seller-based split, §2.2) |
| `resolution` | int | **Target**: 0 = легитимный, 1 = контрафакт |

### Текстовые модальности
| Колонка | Тип | Описание |
|---------|-----|----------|
| `brand_name` | str | Название бренда |
| `name_rus` | str | Название товара |
| `description` | str | Описание товара |
| `CommercialTypeName4` | str | Категория |

### Поведенческие метаданные продавца и товара
| Группа | Колонки | Описание |
|--------|---------|----------|
| Рейтинги | `rating_1_count`..`rating_5_count` | Распределение оценок 1–5 ⭐ |
| Контент | `comments_published_count`, `photos_published_count`, `videos_published_count` | Объём пользовательского контента |
| Цена | `PriceDiscounted` | Цена со скидкой |
| Срок жизни | `item_time_alive`, `seller_time_alive` | Возраст карточки и продавца в днях |
| Fake returns | `item_count_fake_returns7/30/90` | Мошеннические возвраты за окно |
| Sales | `item_count_sales7/30/90`, `OrderAcceptedCountTotal7/30/90` | Динамика продаж |
| Returns | `item_count_returns7/30/90`, `ExemplarReturnedCountTotal7/30/90` | Динамика возвратов |
| GMV | `GmvTotal7/30/90` | Оборот за окно |
| Ассортимент | `ItemVarietyCount`, `ItemAvailableCount` | Разнообразие и доступность |

## Структура `image_embeddings.csv`

CLIP ViT-B/32 эмбеддинги изображений (32-мерные после PCA для baseline B3).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `Unnamed: 0` | int | Индекс |
| `image_name` | str | Имя файла изображения (`{ItemID}.jpg`) |
| `embedding` | str/list | 32-мерный CLIP-эмбеддинг (сериализован) |

## Структура `clip_embeddings.parquet`

Полные 512-мерные CLIP-эмбеддинги для модели Exp-Combined (§3.7).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `ItemID` | int | ID товара |
| `embedding` | list[float] | 512-мерный CLIP ViT-B/32 эмбеддинг |

## Изображения `ml_ozon_counterfeit_test_images/`

Папка содержит исходные PNG-изображения товарных карточек тестовой выборки в формате:

```
ml_ozon_counterfeit_test_images/
├── 10.png
├── 100.png
├── ...
└── 999999.png   # имя файла = ItemID
```

- Формат: PNG, переменное разрешение
- Кол-во: 11 851 файл, ~1,8 GB
- Имя файла: `{ItemID}.png` (соответствует колонке `ItemID` в `ozon_train.csv`)

**Использование**:

```python
from PIL import Image
img = Image.open(f"data/raw/ml_ozon_counterfeit_test_images/{item_id}.png")
```

Изображения нужны только для:
- Воспроизведения вычисления CLIP-эмбеддингов с нуля
- Визуального анализа конкретных кейсов (false positives/negatives)
- Дообучения visual encoder'ов

Для большинства экспериментов достаточно **готовых эмбеддингов** (`image_embeddings.csv` или `clip_embeddings.parquet`) — это в 8× меньше по объёму и не требует GPU.

## Лицензия и условия использования

Датасет предоставлен организаторами **Ozon eCup 2025** в рамках публичного соревнования. Использование — только для академических и исследовательских целей в соответствии с условиями соревнования. Перепубликация и коммерческое использование требуют согласования с правообладателем (ООО «Озон»).

## Этический аспект и анонимизация

- Все идентификаторы (`ItemID`, `SellerID`) обезличены организатором.
- Изображения товаров публично доступны на маркетплейсе.
- Текстовые поля (`description`, `name_rus`) являются публичной частью товарных карточек.
- Персональные данные покупателей в датасете отсутствуют.

## Воспроизводимость

- Разбиение train/test зафиксировано через `random_state=42` и `seller-based split` (§2.2 диссертации).
- Скрипты препроцессинга и обучения — в корневых ноутбуках репозитория.
- Артефакты моделей и метрики — в папке `artifacts/` (см. структуру в Приложении А).
